Overview

I participated in the project advised by professor Mark W. Newman, and helped one of his PhD student Chuan-Che Huang as a research assistant.

 

Details

 

My main responsibilities were building a smart home hub with SmartThings sensors, writing codes to grab data from the hub, desigining a schema for storing data, and dumping data into Firebase database.

 

For this project, I used three main sensors - Aeon Multisensor, SmartThings Contact sensor, and SmartThings Smart Outlet. From these sensors, I could get information about both real-time(temperature, humidity, illuminance) and event-based(door status, outlet status) data.

 

After setting up all sensors, I could access smart home hub and grab all sensor data through Groovy code. Then I came up with database schema to store data. The database has three main branches - events, timeseries, and metrics. Event branch is for storing event-based sensor(Contact Sensor, Smart Outlet) data, timereries branch is for time-based sensor(Aeon Multisensor) data, and metrics branch is for storing metadata of each household and user.

 

Further Step

For the next step, this project is about to launch a mobile application for end-users. This application grabs data from Firebase database and show them to users, and it also lets users to modify their temperature settings based on the data.
