Overview

I developed a django web application that helps University of Michigan, School of Information Masters students to plan their semester schedules. In the application, students can see the availability of each course in each semester as they choose courses they are considering to take in the future.

 

Details

University of Michigan, School of Information introduced a new curriculum in 2017, which is based on 'mastery courses' that have many pre-requisite courses. This structure required students to plan backwards from the mastery course they are considering, and many aspects made the task more complicated: number of total credits, constraints on courses, different offerings per semester, multiple choice pre-requisites, and so on.

In order to handle this complexity, I implemented a top-down algorithm, letting users select a mastery course to reduce the number of options. I also enhanced the usability of the application by changing the status of each course real-time based on user inputs. For example, certain courses turn into 'available' only if a user selects all pre-requisite courses.
